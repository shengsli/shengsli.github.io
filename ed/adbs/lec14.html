5 optimistic concurrency control, OCC

5.1 3 phases

  1. read phase
    • write to private/local workspace
  2. validation phase
    • check all previous txns, whether conflict or not
  3. write phase
    • apply its local copy to database

5.2 occ example

  • read, validate, write are generate by dbms
  • only track write timestamp
    • why? in local workspace, no dirty values in database
  • difference with timestamp cc: <2019-03-18 Mon 09:31> watch recording
    • timestamp is assigned at the beginning of validation phase
  • destroy local copy once commit

5.3 validation phase

  1. Ti copmletes all 3 phases before Tj begins. -> no overlap
  2. Tj does not write to any object read by Ti
