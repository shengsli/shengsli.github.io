2 transcations (txn)

2.1 user perspective

  • txn is a group of operations
    • commit or abort (rollback)

2.2 example: txn

BEGIN
# operations
END
  • consistent db before and after txn
  • when executing txn, db is temporarily inconsistent

2.3 db perspective

  • txn is a sequence of read/write operations.
    • ignore anything else
  • db is a fixed set of named data objects.

2.4 strawman execution

  • no interleaving of txn
  • before a txn starts, make a copy of the entire database
    • failure: power failure, crash
    • if txn want to abort
    • used by sql-lite, designed for embedded systems, mobile phones
