1 Jargon

  • DBMS stores a database as one or more files on disk.
  • a file is (either a collection of records or) a collection of pages.
  • pages contain records
  • a heap file is an unordered collection of pages where records are stored in no particular order.
    • 2 implementation: linked list, page directory

2 Context

  • pages stored on disk -> cache pages into main memory
  • records are stored in pages
  • pages are organized into logical collections or files
  • notes:
    • a collection of records is not limited to the contents of a single page
    • a file can span several pages
  • In this section, we consider how a collection of pages can be organized as a file.

3 Implementing Heap Files

  • a heap file is an unordered set of records.
  • data in pages of a heal file is not ordered.
  • every record in the file has a unique rid.
  • every page in a file is of the same size.
  • operations:
    • create(file)
    • destory(file)
    • insert(record)
    • delete(rid)
    • get(rid)
    • scan() all records
  • requirements:
Table 1: requirements
functions requirements
scans keep track of pages in each heap file
efficient insert keep track of pages that contain free space
  • 2 alternative ways:
    • pages must hold 2 pointers which are page ids for file-level bookkeeping in addition to data.

3.1 Implementation 1: Linked List of Pages

3.1.1 how to implement?

  • maintain a heap file as a doubly linked list of pages
  • DBMS can remember there 1st page is located
    • by maintaining a table containing pairs of <heap_file_name,page_laddr>
  • header page: 1st page of the file
  • maintain information of empty slots created by deletion -> 2 distinct parts:
    • how to keep track of free space within a page?
      • ans: DMS sec9.6
    • how to keep track of pages that have some free space?
      • ans: maintain a doubly linked list of pages with free space
      • and a doubly linked list with full space
  • remarks: each pointer is really a page id.

3.1.2 how it works?

  • require a new page:
    • make a request to disk space manager
    • add to the list of pages in the file
  • delete a page:
    • remove the page from the list
    • disk space manager deallocate it
  • remarks:
    • this scheme can be easily generalize to allocate or deallocate a sequence of several pages
    • disadvantages:
      • virtually all pages in a file will be on the free list if records are of variable length.
        • because it is likely that ever page has a leas a few free bytes.
      • to insert a typical record, we must retrieve and examine several pages on the free list before we find one with enough free space.
    • directory-based heap file addresses disadvantages.

3.2 Implementation 2: Directory of Pages

3.2.1 extract from lec03 p9

  • DBMS maintains special pages (directory of pages) that tracks the location of data pages in the database file.
  • each directory entry identifies a page
  • the directory records free space per page
    • efficient free space search
    • granularity as trade-off space vs. accuracy
    • disadvantages: hosting the directory introduces memory overhead

3.2.2

  • maintain a directory of pages
  • DBMS must remember where 1st directory page of each heap file is located
  • the directory itself is a collection of pages and is shown as a linked list.
  • each directory entry identifies a page (or a sequence of pages) in the heap file.
  • as the heap file grows or shrinks, # of entries in the directories grows or shrinks correspondingly.
  • remarks:
    • each directory entry is quite small cf. a typical page.
    • the size of directory is likely to be very small cf. the size of the heap file.
  • free space can be managed by
    • maintaining a bit per entry, indicating whether the page has any free space
    • or a count per entry, indicating the amount of free space.
  • if the file contains variable-length records,
    • we can examine the free space count for an entry to determine if the record fits on the page pointed to by the entry.
  • since several entries fit on a directory page, we can efficiently search for a data page with enough space to hold a record to be inserted.
