  • suitable for equality selections

3 Static hashing schemes

3.1 Chained hashing

  • Hash index is a collection of buckets.
  • build static hash index on column A

3.1.1 Static hash table

  • Bucket = primary page + zero or more overflow pages
  • Buckets contain index data entries

3.1.2 Hash table management

  • Operations:
    • compute h(r)
    • access primary bucket page
    • search/insert/delete
    • overflow if needed
  • Disadvantage: Long overflow chains can degrade performance
    • operation costs become non-uniform and unpredictable
      • to reduce this problem, h needs to scatter search keys evenly across [0,…,N-1]
      • but large # of entries can still cause long chains
      • dynamic hashing to fix this

3.2 Linear probing

3.2.1 unique keys

  • Single giant table of slots
  • If collision, search for the next free slot
    • search: h(r) to a location and scan for record
  • good locality of reference
  • operations: search, insert, delete: \(O(1)\) when \(load factor < 1\)
    • Load factor is a statistic for hash table.
    • \(load factor = \frac{\#occupied entries}{\#buckets}\)
  • circular buffer
  • operations:
    • search
    • delete:
      • if h(r) is exactly the ideal position, empty the slot
      • else
  • variance of operation cost is high

3.2.2 non-unique keys

  • 2 approches:
    • separate linked list: store values in separate storage area for each key
      • preferable because number of keys in hash table is smaller
    • redundant keys: store duplicate key entries

3.3 Robin hood hashing

Variant of linear probe hashing Distance: # of slots away from its ideal position Key with larger distance is the poor key Poor key steals slots from rich key

3.4 Cuckoo hashing

  • Use multiple hash tables with different hash functions
    • insert:
      1. check every table and pick anyone that has a free slot
      2. if no table has a free slot,
        • evit the element from one of them
        • and then re-hash it to find a new location
    • search
    • delete
  • Operations:
    • search: \(O(1)\) because only one location per hash table is checked
    • insert: \(O(1)\) because only one location per hash table is checked
    • delete: \(O(1)\) if \(load factor < 1/2\)
  • if getstuck in an infinite loop, rebuild the entire hash table with new hash functions
    • 2 hash functions, rebuild at 50% full
    • 3 hash functions, rebuild at 90% full

4 Dynamic hashing schemes

4.1 Extendible hashing p37

  • global depth/global counter: how many bits of hash value need to be inspected to determine in which bucket data entry will be stored
  • local depth/local counter: the least significant bits to determine which busket the entry belongs to

4.2 Linear hashing p47, textbook p379

  • handle long overflow chains without usnig a directory
  • use a family of hash functions $h0, h1, h2, …$
    • N: initial # of buckets
    • h has range [0, N-1]
    • \(h_1 (key) = h(key) mod (2)\)
    • \(h_{i+1}\) doubles the range of \(h_i\)
  • maintain a pointer that tracks the next bucket to split
  • when any bucket overflows, split the bucket at the pointer location
  • split criterion is left up to the implementation
    • space utilization of a bucket beyond % capacity
    • or, average length of overflow chains longer than p pages
