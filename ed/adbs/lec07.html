5 Sorting algorithm

  • if data fits in memory, use standard a sorting algoorithm, like quick-sort
  • otherwise, write data out to disk, e.g. external sorting

5.1 external sorting

  • how to sort a file of records whose size exceeds the available main memory space? divide and conquer:
    • sort chunks of data that fit in memory
    • write back sorted chunks to disk
    • combined sorted chunks into a single large file
  • approach the task in two phases:
    1. sort a file of arbitrary size is possible using only 3 buffer pages
    2. refine this algorithm to make effective use of larger buffer sizes

5.2 2-way external merge sort (B=3 buffer pages) p9

  • Context:
    • files are broken into N pages.
    • DBMS has a finite number of B fixed-size buffer pages.
  • 2 phases:
    • pass #0:
      • read a page into memory, sort it, and write it back to disk (uses 1 buffer page)
      • each sorted set od pages is called a run
    • pass #1, #2, …
      • recursively merge pairs of runs into twice as long (uses 3 buffer pages)
  • \(\#passes = 1 + log_2 N\)
  • total I/O cost \(=2N\cdot \#passes\)

5.3 External merge sort (\(B>3\) bufer pages)

  • how to effectively use full buffer space during in-memory sort?
    • reduce # of initial runs by using the full buffer space during in-memory sort
    • reduce # of passes by merging B-1 runs at a time
  • 2 phases:
    • pass #0:
      • use B buffer pages
      • produce N/B sorted runs of size B
    • pass #1, #2, …
      • merge B-1 runs (i.e. multi-way merge)
  • \(\#passes = 1 + log_{B-1}{(N/B)}\)
  • total I/O cost \(=2N\cdot \#passes\)

6.2 What is bulk-loading?

  • Entries are added to a B+ tree in two ways.
    1. a collection of data records with a B+ tree index on it
      • add a record to the collection
    2. a collection of data records for which we want to create a B+ tree index on some key fields
      • start with an empty tree and insert an entry for each data record, one at a time
  • This approach is expensive because each entry requires us to start from the root and go down to the appropriate leaf page. Even though the index-level pages are likely to stay in the appropriate leaf page.
  • For this reason, many systems provide a buil-loading utility for creating a B+ tree index on an existing collection on an existing collection of data records.
    1. sort data entries k* to be inserted into the (to be created) B+ tree according to saerch key k.
    2. if the entries are key-pointer pairs, sorting them does not mean soring the data records that ar epointd to.
