9 basic idea: logging

  • record undo and redo information in a log file
    • e.g. update, abort, commit, …
  • if crash, replay all actions to come up the same state in memory

10 write-ahead logging (WAL)

  • before making changes, make a record in log file
  • 2 important properties:
    1. write to log, then write to disk
    2. flash all log entries to memory, then write to disk
      • first write intention, then do the action

11 log records

  • each log record has a unique log sequence number (LSN)
  • log record fields: LSN + txn ID + type
    • type: BEGIN, COMMIT, ABORT, END, …
