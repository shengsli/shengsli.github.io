1 cost estimation

  • assumption: attribute values have a uniform distribution
  • in practice, non-uniform distribution.
  • solution: store the frequency of attribute values.
  • problem: domain of an attribute is huge.
  • solution 2: store histogram

2 histograms

2.1 equi-width histograms

  • problem: domain can be large -> too many buckets if keep a bucket for each attribute value
    • require more memory
  • solution: fix # of buckets to B, divide attribute domain into B buckets

2.1.1 equality selection

  • inside each bucket, attribute values are uniformly distributed.

2.2 equi-depth histograms

  • size of bucket is variable
  • each bucket has the same height
  • high-frequency values have smaller bucket, have better estimate
  • round-up if cannot divide equally

7 join tree shapes

    • left-deep plan
