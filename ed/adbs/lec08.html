2 query evaluation

  • a query as a large monolithic block is broken into smaller pieces of query operators
    • operators from rational algebra are arranged in a (syntax) tree called query plan
    • it is a tree of ops
      • edge: is flow of data, data flows from leaves to the root
      • root: will be the result/output of query
      • node: RA op
  • relational algebra: express the op tree
  • next monday: query optimization

3 query evaluation ops

  • different implementations of an RA op
  • each variant implementation of RA op is a physical operator.
  • physical operators exploit properties, such as:
    • presence or absence of indexes on the input files
    • sortedness and size of the input files
    • space in the buffer pool, buffer replacement policy

4 query evaluation plan

  • logical -> physical
  • query optimization = choose best physical plan

5 workflow:

  1. parse given query
  2. translate query to RA
  3. enumerate physical plans by selecting physical ops and order of operators
  4. decide physical query plan by cost model. which is better?
    • these are all estimated cost in theory, it may fail
  5. select the optimal query plan
  1. sql query
  2. logical query plan
  3. physical query plan
  4. physical query plan + estimated cost
  5. optimal query plan
