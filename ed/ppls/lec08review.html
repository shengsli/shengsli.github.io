2 example: build a monitor p96 pseudocode

  • elements:
    • a circular bounded buffer
    • front and rear indices (same as semaphore-version producers and consumers in lec07)
    • count
    • condition variables: cond not_full, not_empty
    • operations/procedures: deposit, fetch
  • NOTE: wait and signal on semaphores are not quite the same as wait and signal in monitors.
    • In monitors, signal not remember the state of semaphore variable.
  • If the buffer is empty, consumers wait/suspend. Otherwise, carry on. => need not_full
  • If the buffer is full, producers wait/suspend. Otherwise, carry on. => need not_empty

3 building blocks

3.1 mutual exclusion

  • From an oop perspective, a monitor is like an object which encapsulates some data to which access is only permitted through a set of methods.
  • A built-in feature of monitors is that at most one thread is active within the monitor at any one time.
  • wait and signal allow threads giving up controls/locks over monitor objects for other threads.
