1 communicators

  • communicator allows to define new groups dynamically
  • example: sketch a structure of divide and conquer
  • mpi comm world: global communnicator
  • create new communicators from old ones by collectively calling:
    • MPI_Comm_split(MPI_Comm old, int colour, int key, MPI_Comm *new)
      • old: can remember old group
      • colour: control of how that communicator takes place
      • key: how indices are assigned
      • new: new group, different number indicating different groups
  • SPMD mode:
  • color: all processes that call the same color become a subgroup
  • key: once a process is assigned to a new communicator, how ids within that new communicator are assigned
    • those keys are used, in each new subgroup, to sort processes into the process order

1.1 divide and conquer communicators p170

  • divide and then unwind back
  • divide and conquer is based on process id rather than data.
  • case: google hyper quicksort
  • MPI_Comm_split(..., myrank<(p/2), ...)
    • lower indexes have the same ranks
    • higher indexes keep the same order
