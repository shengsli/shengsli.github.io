1 mpi task farm

  • don't know which task is easy or hard
  • farmer control task distribution
  • phases:
    1. give one task to every worker
    2. demand-driven
    3. join the result and exit
  • mpi (c programming) often require 3 things
    1. pointer
    2. type
    3. size
  1. MPI_Send
    • dest: process id
    • tag can be task index, farmer can track which task the worker is doing
  2. MPI_Recv
    • source
    • tag: wild card
    • status: MIISOURCE, MPITAG
  • race condition: MPIRecv
  • wild cards: magic no.
  • multiple processes end with same tag: messages come in order
