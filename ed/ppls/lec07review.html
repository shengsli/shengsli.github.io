1 Big picture

  • Today we are gonna change our implementation style.
  • The mechanisms we have designed so far all have been implemented directly in the user-address space.
    • probably hidden inside a library, but nonetheless invisible to the OS.
    • e.g. spinning version of lock implementation
    • e.g. looping in barrier implementation
  • drawback:
    • depend on the system that the program is running on and what is running at the same time
    • when threads are waiting. they are taking up CPU cycles, e.g. reads when waiting lock changes.
  • We will look at techniques which avoid resource wastage problem by having help of OS.
    • e.g. OS suspends and reactivates/resumes threads
    • provide the same logical behaviours like lock and barrier without overhead of spinning

3 semaphores (dutch)

  • is a special shared variable hold integer value
  • 2 atomic operations:
    1. P(s): <await (s>0) s=s-1;>: Probeer (try). decrement only when s is positive, like probing
    2. V(s): <s=s+1;>: Verhoog (increase). signal
  • binary semaphore: only ever takes value of 0 or 1

4 use semaphores to implement critical section p79

sem mutex = 1;
co [i = 1 to n] {
  while (whatever) {
    P(mutex);
    critical section;
    V(mutex);
    non-critical section;
  }
}

7 version 1: single item buffer which allows multiple producers and consumers

7.1 3 things

  1. sem full: counts how many space in buffer is full. (in version 1, buffer only has 1 space)
  2. sem empty: counts how many space in buffer is empty.
  3. buffer

7.2 remarks

  • full and empty count resources
  • requirement of stopping producers is based on # of empty spaces
    • producers progress if buffer is empty.
  • requirement of stopping consumers is based on # of full spaces
    • consumers progress if buffer is full.
  • producers carry on when empty is non-negative
  • consumers carry on when full is non-negative
  • We can't have blocking behaviour without 2 semaphores.
  • Only one of the semaphores will ever be 1. aka split binary semaphore
  • generalization: counting semaphore

7.3 code in co-notation p82

T buf; sem empty = 1, full = 0;
co ## producer
  co [j = 1 to M] {
    while (whatever) {
      ... produce new data locally
      P(empty);
      buf = data;
      V(full);
    }
  }
// ## consumer
  co [j = 1 to N] {
    while (whatever) {
      P(full);      ## wait until buf has item
      result = buf;
      V(empty);
      ... handle result locally
    }
  }

8 TODO version 2: multi-space buffer, recording 30:00

  • bounded buffer: an array with 2 integer indices (front and rear)
  • circular buffer: used spaces are always contiguous
    • producers P(empty)
    • consumers V(full)
    • producers produce at the rear
    • consumers consume at the front
T buf[n]; int front = 0, rear = 0;
sem empty = n, full = 0;
co ## producer
  while (whatever) {
    ... produce new data locally
    P(empty);
    buf[rear] = data;
    rear = (rear + 1) % n;
    V(full);
  }
// ## consumer
  while (whatever) {
    P(full);
    result = buf[front];
    front = (front + 1) % n;
    V(empty);
    ... handle result locally
  }
}

9 TODO version 3

9.1 extra issue with version 2

  • race conditions: increment of rear and front

9.2 solution: 2 levels of protection

  1. use a split counting semaphore to avoid buffer overflow (or underflow)
  2. use 2 mutual exclusion semaphores to prevent interference between producers (and another to prevent interference between consumers).
    • This only allows up to 1 consumer and 1 producer to be actively simultaneously within a non-empty, non-full buffer.
T buf[n]; int front = 0, rear = 0;
sem empty = n, full = 0, mutexP = 1, mutexC = 1;
co
  co [i = 1 to M] {
    while (whatever) {
      ...produce new data locally
      P(empty);
        P(mutexP); buf[rear] = data; rear = (rear + 1) % n; V(mutexP);
      V(full);
    }
  }
//
  co [j = 1 to N] {
    while (whatever) {
      P(full);
        P(mutexC); result = buf[front]; front = (front + 1) % n; V(mutexC);
      V(empty);
      ... handle result locally
    } 
  }
