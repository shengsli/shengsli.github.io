This lecture is about distributed memory programming

  • no more shared memory or no shared variables
  • need some other mechanisms beyond reads and writes, e.g. message passing
  • There are several approaches to abstracting and implementing such a model. We will focus on message passing, which dominates the performance-oriented parallel computing world.

1 Programming with message passing

  • send and receive primitives for transferring data out of and into local address spaces
  • We express algorithms as a collection of local perspectives.
  • These are often captured in a single program source using Single Program Multiple Data (SMPD) style, with different processes following different paths through the same code, branching with respect to local data values and/or to some process identifier.

2 Collective operations

operation  
broadcast everyone gets a copy of the same values
scatter partition data and spread across to group
gather gather data from across the group
reduction gather values with an associative operation
scan reduce and compute all the ordered partial reduction

3 MPI concepts, in a nutshell

  • creates processes statically using mpirun or spawn processes dynamically

3.1 communicator

  • All communications take place within the context of "communication spaces", called communicators, which denotes sets of processes.
  • A process can belong to many communicators simultaneously.
  • New communicator can be defined dynamically.

3.2 Send/Receive

  • send must specify a target.
  • receive can wild card on matching order.
  • Messages can be tagged with an extra value to aid disambiguation.
  • There are many synchronization modes and collective operations.

4 Example: Hello World in MPI

int main(int argc, char *argv)
{
    int rank, p;
    MPI_Init(&argc, &argv);
    MPI_Comm_rank(MPI_COMM_WORLD, &RANK);
    MPI_Comm_Size(MPI_COMM_WORLD, &P);
    printf("Hello world from process %d of %d\n", rank, p);
    MPI_Finalize();
}

5 MPI primitives

  • int MPI_Init(int *argc, char ***argv) and int MPI_Finalize() must be called once by every participating process before any other MPI calls.
  • Each process has a unique identifier in each communicator of which it is a member (range 0..members-1). The built-in communicator to which all processes belong is called MPI_COMM_WORLD. A process can find the size of a communicator and its own rank within it by int MPI_Comm_Size(MPI_Comm comm, int *np) and int MPI_Comm_rank(MPI_Comm comm, int *me).
