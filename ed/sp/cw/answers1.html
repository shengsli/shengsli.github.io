1.1 Similarity and Difference of Meltdown and Spectre

  • Similarities:
    • Both exploits speculative execution.
    • Both breaks the isolation between memory spaces.
  • Differences:
    • Meltdown breaks the isolation between kernel space and user space. Spectre breaks the isolation between applications' memory spaces.
    • Meltdown exploits privilege escalation on Intel processors, caused by out-of-order execution. Meltdown does not exploits branch prediction. Spectre exploits branch prediction. Spectre also works on non-Intel processors.

  • It breaks confidentiality and integirty.

    • integrity: kernel space integrity is compromised.

1.4 Possible consequences and how attackers make use of them

  • possible consequences:
    • meltdown: break the isolation between kernel space and user space
    • spectre: break the isolation between applications
  • how attackers make use of them:
    • meltdown: allows attackers to read kernel memory from user space or dump the entire memory by reading virtual address
    • spectre: allows attackers to access victims' user address space

1.5 Differences between hardware and software vulnerabilities

  • Hardware vulnerabilities are hardware flaws that can be exploited by attackers.
  • Software vulnerabilities are weakness at software level which might be fixed without redesign hardware.
  • The way I draw the distinction between them is whether the vulnerability is caused by hardware or software flaws.
  • For example, CVE-2019-9787 is a software vulnerability. This vulnerability is WordPress before 5.1.1 does not properly filter comment content. Attackers can execute malicious code by CSRF. This vulnerability belongs to CWE-352. Possible fix can be done by carefully filtering comments. I suppose the latest WordPress already handles it.

1.6 Mr. Super Secure

It is possible that firewall or anti-virus might help prevent or mitigate some attacks that exploits meltdown or spectre, but I don't think they can fully prevent exploitation of meltdown and spectre.

  • Meltdown and spectre are hardware vulnerabilities that reside in processors. Patches or other prevention at software level can help but can never solve the problem at its root. Meltdown and spectre undermine security features built into processors by exploiting the whole concept of speculative execution.
  • firewall: It is unlikely that you can know what what IP addresses are malicious.
  • anti-virus: For meltdown, it is possible that anti-virus software can detect malicious behaviours which try accessing kernel memory. But there is a fat chance that users authorise these programs by clicking Yes. Because it is unlikely that users know whether it is a legitimate or a malicious request. Same principles apply to spectre. For example, on browsers, one site can access another site's credential cookie. Anti-virus might be able to detect one application tries to access another's memory, or blacklist some websites, but again it is very hard to detect illegal access between applications in user space.

1.7 Ways to protect from attacks to meltdown and spectre

  • keep software up-to-date
  • only visit encrypted and trusted sites
  • reduce unnecessary features, services or software. More features expose more vulnerabilities.
