  • 3.1 OpenSSH Configuration
    • a) Only allow user to login through ssh:
      AllowUsers user
    • b) Disable password authentication and only allow user to login with a private key:
      ssh-keygen -t rsa # generate rsa key
ssh-copy-id -i id_rsa.pub user@localhost -p 2222 # copy public key to server
ssh -p 2222 user@localhost # login, enter passphrase if required
      • I used RSA to generate keys instead of DSA, because:
        • DSA makes use of factorization of large integer. RSA takes advantage of discrete logarithm algorithm.
        • DSA is faster for signature generation but slower for validation, slower when encrypting but faster when decrypting.
        • Given that security is considered equivalent with the equal key length, I favour the faster one.
      • A complete demo is attached to appendices.
